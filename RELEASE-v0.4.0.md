# API Switch v0.4.0

## 🎯 核心亮点

### 🔘 托盘右键快捷切换模型

右键系统托盘图标，可直接选择模型发起对话，无需打开主窗口。

### 🔄 渠道自动校对

添加/编辑渠道时，一键"拉取模型"自动完成 API 类型检测、Base URL 校对、多协议 Fallback 拉取。支持 CODING PLAN 等中转站的模型发现与展示。

### 📋 智能模型预选

拉取模型后自动选中 6 个月内发布的新模型 + 当前渠道已有模型，新增条目默认开启。减少手动勾选，开箱即用。

### ⚡ 渠道 & 模型一键测速

- **渠道测速**：测试所有渠道的 URL 可达性与延迟，逐个显示结果，列表不跳动
- **模型测速**：对所有 API 池模型发送真实请求，成功显示绿色响应时间 `(1.2s)`，失败显示红色 ✗
- 帮你快速排除不可用的渠道和模型

### 🛡️ 熔断优化

- 自动禁用不可恢复模型：收到 401/403/410 等状态码时自动关闭模型（可在设置中自定义）
- 模型冷却机制：失败自动冷却 300s，成功自动恢复，冷却中的模型不参与路由
- 恢复等待时间改为滑块控件（300-1800s），默认 600s

### 🌐 中英双语使用指南

使用指南按界面语言自动切换：中文 → `GUIDE_CN.md`，英文 → `GUIDE.md`。

## 📝 完整更新内容

### 新功能

| 功能 | 说明 |
|------|------|
| API 池一键测速 | 逐个测试所有模型延迟，成功显示绿色时间，失败显示红色 ✗，结果持久化保存 |
| 渠道一键测速 | 测试所有渠道可达性（不限于已启用），本地实时回填，列表不跳动 |
| 智能模型预选 | 自动选中 6 个月内新模型 + 当前渠道已有模型 |
| 渠道空时自动弹窗 | 进入渠道页无渠道时自动弹出添加对话框 |
| 使用指南双语 | 按语言自动切换中文/英文 GUIDE |
| 恢复等待滑块 | 冷却恢复时间改为滑块控件（300-1800s），默认 600s |

### 体验优化

| 项目 | 说明 |
|------|------|
| 渠道列表 nowrap | 状态、响应时间、模型数列防换行 |
| 渠道保存缓存同步 | 保存后同时刷新 entries 和 channels 缓存 |
| 模型选择同步修复 | 清空所有选择后保存能正确删除已有条目 |
| 新增模型默认开启 | sync_entries 新建条目 enabled=1 |
| 渠道自动校对 | 单按钮智能拉模型，自动检测 API 类型与 URL，支持多种协议 Fallback |
| URL/类型自动回填 | 校对成功后自动回填 api_type 与 base_url |
| API 池模型目录 | 本地 models.json 索引，实时显示模型元信息（发布日期、能力、上下文长度等） |
| 近似模型匹配 | 支持 provider/model 前缀、-free/-preview 等后缀清洗与相似度匹配 |
| 自动禁用生效 | 正式代理收到命中状态码时自动 disabled=false |
| 测试对话修复 | 请求序号隔离，关闭不卡住后续测试 |
| 日志尝试路径修复 | attempt_path 正确格式化显示 |

### 技术细节

| 项目 | 说明 |
|------|------|
| 数据库迁移 | `api_entries` 新增 `response_ms TEXT DEFAULT ''`，启动自动补齐 |
| 新增 Tauri 命令 | `test_entry_latency`、`update_entry_response_ms` |
| 向下兼容 | 所有变更完全向后兼容，老数据库自动迁移 |
